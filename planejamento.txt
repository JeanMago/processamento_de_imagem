1. Introdução e Escopo do Projeto
1.1. Objetivo Geral
O presente projeto tem como objetivo projetar e desenvolver um software interativo para o Processamento Digital de Imagens (PDI). A aplicação fornecerá uma Interface Gráfica de Utilizador (GUI) que permitirá a manipulação de matrizes de imagens no domínio espacial, aplicando técnicas de realce, filtragem e análise estatística (histogramas) em tempo real.
1.2. Justificativa Teórica e Tecnológica
O processamento de imagens consiste na aplicação de algoritmos matemáticos sobre matrizes bidimensionais cujos elementos (píxeis) representam níveis de intensidade luminosa. Conforme a literatura estabelecida (AZEVEDO; CONCI; LETA, 2022; GONZALEZ; WOODS, 2010), a visualização interativa dos efeitos das operações ponto a ponto e espaciais é fundamental para a compreensão das características morfológicas da imagem. O sistema será desenvolvido em Python, utilizando a biblioteca NumPy para operações matriciais otimizadas de baixo nível e OpenCV para a implementação eficiente dos algoritmos de visão computacional.
2. Especificação da Interface Gráfica (Wireframe e UX)
A interface foi projetada visando a usabilidade e a resposta imediata (arquitetura event-driven). O layout divide a aplicação em áreas lógicas de controlo, visualização e análise de dados.

4. Requisitos do Sistema
4.1. Requisitos Funcionais (RF)
RF01: O sistema deve importar imagens nos formatos .jpg, .jpeg e .png.
RF02: O sistema deve exportar a imagem processada para o disco local.
RF03: O sistema deve aplicar os algoritmos de Brilho/Contraste, Gaussiano, Sobel e Equalização.
RF04: O sistema deve gerar e desenhar o histograma de frequências da imagem atual no momento exato de qualquer modificação.
RF05: O sistema deve manter o estado original da matriz da imagem para permitir a função de reversão ("Desfazer").
4.2. Requisitos Não Funcionais (RNF)
RNF01: A aplicação deve ser desenvolvida sob o paradigma de Orientação a Objetos ou programação modular estruturada.
RNF02: O tempo de resposta para a aplicação de transformações lineares não deve exceder 500ms para imagens até resolução Full HD.
5. Arquitetura do Software e Fluxo de Dados
A arquitetura do software será baseada no padrão Model-View-Controller (MVC), garantindo o desacoplamento entre a interface gráfica e os cálculos matriciais intensivos.
View (gui.py): Módulo construído com bibliotecas nativas (como Tkinter) ou frameworks (PyQt/PySide). Gere unicamente o desenho dos widgets e a captura de eventos de input do utilizador.
Model (filtros.py): O "motor" do processamento. Contém funções que recebem objetos numpy.ndarray (matrizes da imagem) e parâmetros escalares numéricos. Encapsula as chamadas diretas às funções da biblioteca OpenCV (ex: cv2.filter2D, cv2.calcHist).
Controller (main.py): Instancia a janela, armazena a variável global/instância da imagem carregada em memória (estado_atual_imagem) e atua como ponte, enviando os parâmetros da View para o Model e devolvendo a matriz calculada para atualização na View.
